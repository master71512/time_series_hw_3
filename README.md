# Исследование влияния трансформаций временных рядов на качество прогнозирования

## Постановка задачи / гипотеза
**Гипотеза:** Оптимальный метод предобработки (трансформации) временного ряда зависит от его типа. Для рядов с разными статистическими свойствами (стационарность, сезонность, волатильность) разные преобразования дают наибольший прирост качества прогноза относительно базовых моделей.

**Цель:** Проверить, для каких типов рядов (кластеров) какие преобразования (log1p, Box-Cox, взятие разности) наиболее эффективны при прогнозировании с помощью модели CatBoost.

## Описание данных
*   **Источник:** M4 dataset
*   **Объем:** Проанализировано 200 временных рядов.
*   **Структура:** Были взяты дневные ряды.


## Методология эксперимента

### Кластеризация рядов
Ряды были разделены на типы с помощью двух подходов:
1.  **Кластеризация по признакам (feature-based):** Использовались статистические характеристики ряда (среднее, дисперсия, коэффициент асимметрии, энтропия, значения ACF, результаты теста ADF). Применен алгоритм KMeans.
2.  **Кластеризация по форме (shape-based):** Использован алгоритм `TimeSeriesKMeans` с метрикой DTW (Dynamic Time Warping), позволяющей сравнивать формы рядов, несмотря на возможные временные сдвиги.

Число кластеров (`N_CLUSTERS`) было выбрано на основе  метода локтя.

### Бейзлайн-модели
Для оценки качества прогнозов использовались простые и классические статистические методы:
*   **Naive:** Прогноз равен последнему наблюдению.
*   **SeasonalNaive:** Прогноз равен значению за предыдущий аналогичный период (с заданной сезонностью).
*   **Theta:** Метод Theta из библиотеки `sktime`.
*   **Auto.ETS:** Автоматическая модель экспоненциального сглаживания.


### Глобальная модель
В качестве основной модели для сравнения преобразований использован **CatBoost** (градиентный бустинг на категориальных признаках). Модель обучалась на лаговых значениях ряда (lag-признаки) для прогнозирования на 1 шаг вперед с итеративным построением прогноза на горизонт 14.

### Трансформации и схема эксперимента
Для каждого ряда:
1.  Ряд делится на обучающую (`train`) и тестовую (`test`) выборки (последние 14 точек).
2.  На обучающей выборке применяются 4 варианта предобработки:
    *   **Raw:** Исходный ряд без преобразований.
    *   **Log1p:** `log(1 + x)`.
    *   **Box-Cox:** Преобразование Бокса-Кокса для стабилизации дисперсии.
    *   **Differencing:** Взятие первой разности (`diff=1`).
3.  На каждом преобразованном ряду обучается модель CatBoost.
4.  Строятся прогнозы на 14 шагов вперед.
5.  Прогнозы восстанавливаются в исходную шкалу (обратные преобразования).
6.  Рассчитываются метрики качества на тестовой выборке в исходной шкале.

**Метрики качества:**
*   **MAE (Mean Absolute Error):** Средняя абсолютная ошибка.
*   **RMSE (Root Mean Squared Error):** Среднеквадратичная ошибка (чувствительна к выбросам).
*   **SMAPE (Symmetric Mean Absolute Percentage Error):** Симметричная относительная ошибка в процентах.
*   **MASE (Mean Absolute Scaled Error):** Масштабированная ошибка, позволяет сравнивать ряды с разными масштабами.

## Выводы

**Гипотеза подтвердилась:** Оптимальный тип преобразования действительно зависит от характеристик временного ряда. Не существует универсальной "лучшей" трансформации.
